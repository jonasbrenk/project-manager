UI Bug Report

UI-01 –

Show Done

/

Hide Done

überlagert Header

Aktion: Show Done oder Hide Done ausführen.

Fehler: Während der Animation schiebt sich Content kurz über die Headerzeile.

Soll: Der Header bleibt jederzeit über dem Content. Kein Element darf ihn während der Animation überdecken.

Fix-Anforderung: Layering und Clipping der Animation korrigieren.

Screenshot: [Link]

UI-02 – Compress-Header klappt zu früh ein

Aktion: In der Ansicht nach oben scrollen.

Fehler: Der obere Header komprimiert schneller als der Links-Header und der Content hochscrollen. Dadurch entsteht ein zu großer Gap.

Soll: Header, Links-Header und Content bewegen sich synchron 1:1.

Fix-Anforderung: Compress-Fortschritt direkt an den Scroll-Fortschritt koppeln. Keine vorauseilende Animation.

Screenshot: [Link]

UI-03 – Swipe-Element wird transparent

Aktion: Ein Element horizontal swipen.

Fehler: Das Element wird teilweise durchsichtig. Die Swipe-Aktionen sind bereits dahinter sichtbar.

Soll: Das Element bleibt vollständig deckend. Aktionen sind nur im freigelegten Bereich sichtbar.

Fix-Anforderung: Transparenz während des Swipes entfernen.

Screenshot: [Link]

UI-04 – Swipe und Scrollen konkurrieren

Aktion: Ein Element leicht diagonal swipen.

Fehler: Die Liste scrollt gleichzeitig vertikal. Die Bewegung wirkt zitterig und unruhig.

Soll: Horizontale Swipe-Geste und vertikales Scrollen werden klar getrennt.

Fix-Anforderung: Direction Lock und sinnvolle Gesture-Thresholds verwenden.

Screenshot: [Link]

UI-05 – Falscher Border Radius bei oberster Swipe Action

Aktion: Swipe Actions beim obersten Element öffnen.

Fehler: Die linke Seite ist abgerundet und wirkt dadurch falsch.

Soll: Die Swipe Action schließt bündig ab.

Fix-Anforderung: Linken Border Radius beim obersten Element entfernen.

Screenshot: [Link]

UI-06 – Move-Popup kollidiert mit Plus-Button

Aktion: Move-Funktion öffnen.

Fehler: Das Popup liegt hinter oder zu nah am Plus-Button.

Soll: Das Popup ist vollständig sichtbar und wird nicht überlagert.

Fix-Anforderung: Popup höher priorisieren und abhängig vom verfügbaren Platz positionieren.

Screenshot: [Link]

UI-07 – Move-Funktion zu eingeschränkt

Aktion: Element in eine Liste verschieben.

Fehler: Es gibt nur Insert at Start und Insert at End.

Soll: Eine konkrete Position innerhalb der Liste muss wählbar sein.

Fix-Anforderung: Einfügen vor/nach einem Element oder Drag-and-Drop ermöglichen.

Screenshot: [Link]

UI-08 – Auswahl-Highlights inkonsistent

Aktion: Elemente für Löschen, Hide, Show oder Project View auswählen.

Fehler: Highlights erscheinen teilweise, fehlen oder verschwinden.

Soll: Jedes ausgewählte Element wird sofort und stabil markiert.

Fix-Anforderung: Einheitliche Auswahlzustände für alle Ansichten und Elementtypen.

Screenshot: [Link]

UI-09 – Links ohne Auswahl-Highlight

Aktion: Link im Auswahl- oder Löschmodus auswählen.

Fehler: Der Link erhält keine sichtbare Markierung.

Soll: Links werden genauso klar markiert wie Tasks und Projekte.

Fix-Anforderung: Auswahlzustand auf die gesamte Link-Zeile anwenden.

Screenshot: [Link]

UI-10 – Mobile Inputs zeigen eckigen Rahmen

Aktion: Auf Mobile ein Projekt oder einen Task erstellen oder bearbeiten.

Fehler: Inputs erhalten einen unerwünschten rechteckigen Rahmen bzw. ein halbes Highlight.

Soll: Nur der definierte abgerundete Input- und Fokuszustand ist sichtbar.

Fix-Anforderung: Native Mobile-Input-Styles überschreiben und konsistenten Fokuszustand verwenden.

Screenshot: [Link]
